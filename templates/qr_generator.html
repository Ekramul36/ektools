A QR code is just a shortcut for typing

The whole appeal of a QR code is skipping the friction of typing something out by hand — a long WiFi password with mixed case and symbols, a lengthy URL, or contact details that would otherwise mean carefully retyping every character on a phone keyboard. A guest scans a code taped near the router and connects instantly instead of asking to be told the password aloud; a poster or business card carries a link straight to a menu, a portfolio, or a payment page without anyone needing to type a single character. The tradeoff is that whatever gets encoded is only as useful as the printed or displayed code staying legible — a WiFi QR code taped up in bright glare or printed too small defeats the purpose just as surely as no code at all, which is part of why size and color contrast matter more than they might first seem.

How a QR code actually encodes information

 

A QR (Quick Response) code is a two-dimensional barcode — instead of encoding data in a single line of varying-width bars like a traditional barcode, it packs data into a square grid of black and white modules, which lets it hold far more information in the same physical space. A phone camera or scanner reads the pattern, decodes it back into the original text string, and then acts on that string — opening it as a URL if it looks like one, offering to connect to a network if it's formatted as WiFi credentials, or simply displaying it as plain text.

Each content type this tool supports is really just a different way of formatting that underlying text string so a scanning device knows how to interpret it. A URL is encoded as-is; an email address becomes a "mailto:" link; a phone number becomes "tel:"; WiFi credentials are packed into a specific structured format (network name, password, and security type) that phones recognize and offer to auto-connect to. Selecting the right type before generating matters precisely because of this — encoding a WiFi password as plain "text" instead of the "WiFi" type will still produce a valid QR code, but a scanning phone will just display the raw string instead of offering to connect automatically.

Common problems people run into

 

The QR code won't scan. The most frequent cause is low contrast — a light color like yellow or pale blue against white doesn't give a scanner enough contrast to lock onto the pattern reliably. Stick to dark colors like black, navy, or dark green, and make sure the displayed or printed code is large enough (at least roughly 2×2 cm when printed) to scan comfortably at arm's length.

Scanning opens the wrong thing, or just shows text. This usually means the wrong content type was selected before generating — for instance, a phone number typed under "Text" instead of "Phone Number" will display as plain text rather than offering to dial. Regenerate using the correct type from the dropdown.

The WiFi QR code doesn't connect automatically. Some older phones or non-native camera apps don't support WiFi-format QR codes at all and will only display the raw text. Confirm the exact network name (case-sensitive) and password were entered correctly, and try the phone's native camera app if a third-party scanner isn't recognizing the format.

Tips for a QR code that always scans cleanly

 

Generate at the largest size you're likely to need rather than the smallest — a QR code can always be displayed smaller than its native resolution without losing scannability, but scaling a small code up tends to blur the fine detail scanners rely on. For anything that will be printed, err toward the 400px option even if it looks larger than necessary on screen.

Before relying on a printed or posted QR code, always test-scan it yourself with a couple of different phones first — what renders fine on a screen doesn't always print or laminate the same way, and catching a scanning issue before a hundred flyers go up saves a lot of hassle later. For a URL, double-check that "https://" is included; some scanners handle a bare domain fine, but including the full protocol avoids any ambiguity.

Frequently asked questions

 

What can I encode in a QR code?

URLs, plain text, email addresses, phone numbers, WiFi credentials, and SMS messages — select the type from the dropdown before generating.

Do QR codes generated here expire?

No. A QR code itself doesn't expire — it will keep working as long as the content it points to, such as a URL, remains valid.

Can I change the color of the QR code?

Yes, use the color picker to set the foreground color before generating. Keep enough contrast against the white background so scanners can read it reliably.

Why won't my phone scan the QR code?

Usually low contrast, a color too close to white, or the code being displayed too small on screen. Regenerate with a larger size and a dark color, or print it at a larger physical size.

Is my data stored when I generate a QR code?

No. The QR code is generated entirely in your browser — the content you type never leaves your device or gets sent to a server.

Can I put a WiFi QR code somewhere public?

Only if you're comfortable with anyone nearby connecting to that network — scanning it grants network access directly, so treat it the same way you'd treat sharing the password aloud.

Quick recap

 

A QR code is just a fast, scannable stand-in for typing something out by hand, and picking the right content type before generating is what makes the difference between a code that just displays text and one that actually opens a link, dials a number, or connects to WiFi automatically. Because everything here runs in your browser, there's nothing to worry about in terms of your content being logged or stored anywhere. Keep contrast dark, size generous for anything printed, and always test-scan before relying on a code in the wild — those three habits cover the vast majority of scanning problems people run into.
