The two kinds of PDF passwords (and why it matters)

Not all PDF passwords do the same job, and knowing which one you're dealing with saves confusion:

  • Open password (user password). Without this, the PDF won't open at all — you'll get a prompt the instant you try to view it. Banks and payslip systems typically use this, often set to something predictable like your date of birth or the last four digits of a PAN or account number.
  • Permissions password (owner password). The file opens freely, but actions like printing, copying text, editing, or filling forms are disabled until this password is supplied. This is common on official certificates and reports where the issuer wants the content viewable but not easily altered.

A single PDF can carry either one, both, or neither. EkZapp's Unlock PDF tool removes whichever type is present once you provide the correct password — the output is a normal, fully permitted PDF either way.

What if you've forgotten the password?

This is the honest limit of what any unlock tool — EkZapp included — can do. Modern PDFs are typically encrypted with AES-128 or AES-256, the same standard used to protect banking and government systems. Guessing a strong password by brute force would take an impractical amount of time even with serious computing power, so no legitimate free tool can "crack" a properly encrypted PDF without knowing the password.

If you're genuinely locked out, a few paths actually work:

  • Check the issuer's convention first. Bank statements, payslips, and exam certificates very often use a predictable pattern — date of birth in DDMMYYYY format, the first four letters of your name plus a number, or the last digits of an account or roll number. The document that originally told you this (a welcome email, portal instructions) is worth a second look.
  • Contact whoever issued the file. Your bank, HR department, or the exam board can resend the document or confirm the password format — this is the reliable route for anything sensitive.
  • Check your own records. If you set the password yourself using a password manager or a note, that's often faster than any technical workaround.

Be cautious of any site or tool that claims to "crack" or "bypass" a PDF password without you supplying it — for a properly encrypted file, that claim is either false or relies on a method you wouldn't want your sensitive documents anywhere near.

How PDF password protection actually works

It helps to understand what's happening under the hood, because it explains both what unlocking can and can't do. When a PDF is password-protected, the file's content streams — text, images, fonts — are encrypted using a symmetric key, and that key is itself wrapped using a value derived from the password you set. The open password and the permissions password, if both are present, unlock different layers of this same structure: the open password is needed just to decrypt the content at all, while the permissions password controls a separate set of flags that PDF readers check before allowing printing, copying, or editing, even after the file is open.

Removing a password doesn't "hack" anything — it's a legitimate operation available to anyone who already holds the correct password. The tool decrypts the content using the password you supply, then re-saves the file with no encryption and no permission flags at all. That's why the process is instant once the password is correct, and why it's mathematically infeasible when it isn't: there's no shortcut around the encryption itself, only the front door of entering the right key.

Common mistakes to avoid

  • Trying random guesses on someone else's file. Beyond the ethical issue, most secure PDFs don't even let you guess repeatedly in a useful way — you'll just waste time. Go to the source instead.
  • Unlocking and then forgetting to keep a protected backup. If the original password mattered — a sensitive bank document, for instance — keep the protected version somewhere and treat the unlocked copy as a working file, not the one you store long-term.
  • Uploading a locked file to a tool that silently fails. Some merge, split, or compress tools don't clearly explain why they can't process a file — if a PDF operation is failing for no obvious reason, check whether the file is password-protected first.
  • Assuming "unlock" and "unprotect from copying" are the same as "decrypt without the password." Every legitimate unlock tool, including EkZapp's, requires the password you already have. If a tool claims otherwise for a properly encrypted file, treat that claim skeptically.

What's the difference between an open password and a permissions password?

An open password (user password) is required just to view the PDF at all — without it, the file won't open. A single PDF can have either one, both, or neither.
