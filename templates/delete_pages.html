Cleaning up a document without starting over

Documents rarely arrive exactly the way you need them — a scanner adds a blank calibration page, a report includes an appendix nobody asked for, or a contract has one page with information that shouldn't go to every recipient. Deleting the unwanted pages and keeping everything else intact is far faster than recreating the entire document from scratch, and it preserves the original formatting of every page that stays.

How page numbering works when deleting

 

Page numbers entered into this tool always refer to the position of a page in the file as uploaded — the first page is 1, the second is 2, and so on, matching what any standard PDF viewer's page counter would show. This matters when deleting a range: entering "3-5" removes the pages currently sitting in positions three through five, not pages labeled "3", "4", and "5" if the document has its own printed page numbers that differ from its actual position (a document with a cover page, for instance, might have "Page 1" printed on what is actually the second file page).

When multiple pages are deleted at once, the remaining pages automatically shift to fill the gaps, so a ten-page document with pages 3 and 7 removed becomes a clean eight-page file with no missing numbers or blank spaces left behind.

Common problems when deleting PDF pages

 

The wrong pages got deleted. This almost always traces back to counting from a document's printed page numbers instead of its actual file position — a cover page, table of contents, or blank divider can shift everything by one or more compared to what's printed on the page itself.

The file won't upload. Confirm the file is a genuine PDF and not a renamed image or Word document — some file managers hide extensions by default, which can make a .docx look like a .pdf at a glance.

Nothing seems to have changed after deleting. Double-check that the downloaded file is the new one and not a cached copy of the original — browsers sometimes reopen a previous download with the same filename instead of the freshly generated one.

Tips for deleting pages accurately

 

Before submitting, open the PDF in any viewer and note the exact page numbers you want gone, counting from the very first page as 1 — this avoids the common mistake of using a document's own printed page numbers instead of its actual file position.

For a large batch of scattered pages, it's easier to list them explicitly (e.g. "2,5,9,14") rather than guessing at ranges, since a single mistyped number in a range can accidentally remove more or fewer pages than intended. Double-checking the downloaded result against your original list takes only a few seconds and avoids having to redo the whole process.

Frequently asked questions

 

Can I delete multiple pages at once?

Yes. Enter multiple page numbers separated by commas, like 1,3,5, or a range like 2-6, to delete several pages in one go.

What is the difference between Delete Pages and Split PDF?

Delete Pages removes the pages you specify and keeps everything else. Split PDF does the opposite: it extracts only the pages you want and discards the rest.

Will my original PDF be changed?

No. A brand new PDF is created with the remaining pages. Your original uploaded file is never modified.

Is this tool free?

Yes, 100% free, with no signup or credit card required.

How do I know which page numbers to enter?

Page numbers refer to the position in the file as it currently stands, starting from 1 for the first page — the same numbering shown in most PDF viewers' page navigator.

What happens if I enter a page number that doesn't exist?

An out-of-range page number is simply ignored, and the rest of the valid page numbers are still processed normally.

What's the maximum file size?

PDF files up to 50MB are supported.
